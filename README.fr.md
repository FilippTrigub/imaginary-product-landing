# Page de destination NovaSphere v2

NovaSphere v2 est une page de destination conceptuelle présentant « l'avenir de l'informatique personnelle » avec une interface holographique, un traitement quantique et une sécurité neuronale. Ce dépôt contient toutes les ressources du site marketing, y compris le balisage, les styles, les scripts et les médias.

## Fonctionnalités

- **Section héros** présentant l'appareil NovaSphere avec des boutons d'appel à l'action
- **Grille de fonctionnalités** mettant en avant l'interface holographique, le traitement quantique, la sécurité neuronale et la batterie infinie
- **Plans tarifaires** avec les niveaux Standard, Pro et Entreprise
- **Narration de la vision produit** avec des statistiques, une feuille de route à long terme et des visuels
- **Section À propos** décrivant la vision de l'équipe
- **Formulaire de contact** pour recueillir les demandes des utilisateurs
- **Pied de page** avec navigation, coordonnées et liens juridiques

## Structure du projet

```
/
├── index.html        # Page d'accueil principale
├── header.html       # Balisage d'en-tête optionnel (non lié par défaut)
├── team.html         # Page d'équipe en tant que placeholder
├── styles.css        # Styles globaux du site
├── script.js         # Fichier JavaScript placeholder
├── images/
│   ├── logo.svg      # Ressource du logo NovaSphere
│   └── product.svg   # Illustration de l'appareil
└── README*.md        # Documentation multilingue
```

## Prise en main

1. Clonez ou téléchargez ce dépôt.
2. Ouvrez `index.html` dans n'importe quel navigateur moderne (ordinateur ou mobile).
3. Veillez à conserver le dossier `images/` au même niveau que les fichiers HTML afin que les ressources se chargent correctement.

> La page est entièrement statique et ne nécessite aucun processus de build ni dépendance externe, à l'exception de la police hébergée par Google (Montserrat).

## Personnalisation

- Mettez à jour le contenu de `index.html` pour refléter le message de votre produit.
- Modifiez la palette de couleurs et la typographie dans `styles.css` pour correspondre à votre identité visuelle.
- Remplacez `images/logo.svg` et `images/product.svg` par vos propres ressources, en conservant les mêmes noms de fichiers ou en mettant à jour les références dans le HTML.
- Ajoutez de l'interactivité en écrivant du code dans `script.js` (actuellement vide).

## Accessibilité et performances

- Structure HTML sémantique avec titres clairs et identifiants de section pour la navigation
- Contrôles de formulaire accompagnés de leurs étiquettes
- Mise en page responsive gérée via Flexbox en CSS
- Ressources externes minimales pour garantir des temps de chargement rapides
