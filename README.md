# NovaSphere v2.0 - Landing Page

A modern, responsive landing page for NovaSphere, a futuristic personal computing device featuring holographic interfaces and advanced computing technologies.

## 🚀 Project Overview

NovaSphere v2.0 is a conceptual product landing page showcasing an imaginary next-generation computing device. The website features a clean, modern design with smooth animations, responsive layouts, and an intuitive user experience.

## ✨ Features

- **Responsive Design**: Fully responsive layout that works seamlessly across desktop, tablet, and mobile devices
- **Modern UI/UX**: Clean, professional design with smooth animations and transitions
- **Product Showcase**: Comprehensive feature highlights and pricing plans
- **Team Page**: Dedicated page showcasing the team behind NovaSphere
- **Enhanced Navigation**: Sticky header with dropdown menus and breadcrumb navigation
- **Contact Form**: Interactive contact form for user inquiries
- **Optimized Performance**: Lightweight HTML/CSS/JavaScript implementation

## 🚀 Getting Started

### Prerequisites
- A modern web browser (Chrome, Firefox, Safari, Edge)
- Optional: Local web server for development

### Installation

1. Clone the repository:
```bash
git clone <repository-url>
cd /vercel/sandbox
```

2. Open the project:
   - **Option 1**: Simply open `index.html` in your web browser
   - **Option 2**: Use a local development server:
     ```bash
     # Using Python 3
     python3 -m http.server 8000
     
     # Using Node.js (http-server)
     npx http-server -p 8000
     
     # Using PHP
     php -S localhost:8000
     ```

3. Navigate to:
   - Main page: `http://localhost:8000/index.html`
   - Team page: `http://localhost:8000/team.html`
   - Header demo: `http://localhost:8000/header.html`

## 📝 Customization

### Updating Content
- Edit HTML files directly to modify text content
- Update `styles.css` for styling changes
- Modify `script.js` for functionality changes

### Adding New Pages
1. Create a new HTML file
2. Copy the header and footer structure from existing pages
3. Add your custom content in between
4. Update navigation links in all pages

### Changing Images
- Replace SVG files in the `/images/` directory
- Update image references in HTML files
- Maintain aspect ratios for best results
