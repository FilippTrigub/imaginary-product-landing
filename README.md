# NovaSphere - The Future of Personal Computing

This repository contains a static landing site for NovaSphere v2 — a fictional product showcasing a holographic personal computing concept.

**This README** summarizes the project, core features shown on the pages, repository layout, and quick instructions to view the site locally.

**Project Summary**
- NovaSphere v2 is a design and prototype landing site demonstrating a futuristic product concept: holographic interfaces, quantum processing, and adaptive AI.
- The site includes a main landing page (`index.html`), an enhanced header/navigation demo (`header.html`), and a team page (`team.html`).

**Key Features (what the pages demonstrate)**
- Hero section and product imagery with call-to-action buttons.
- Feature grid highlighting holographic interface, quantum processing, and neural security.
- Pricing and plan examples with sign-up and sales CTAs.
- Product vision and stats sections for marketing copy.
- Team page with member bios and responsive card layout.
- Enhanced header page demonstrating dropdowns, mobile menu, and breadcrumb navigation.

**Repository Structure**
- `index.html` — Main landing page (hero, features, pricing, contact form).
- `header.html` — Enhanced header/navigation demo (dropdowns, breadcrumbs, mobile menu).
- `team.html` — Team page with bios and responsive grid.
- `styles.css` — Global styles used across pages.
- `script.js` — Small client-side scripts (form handlers, UI interactions).
- `images/` — Image assets and logos.
- `README.md` — This file.

**View Locally**
1. Open the files directly in your browser by double-clicking `index.html` (works for most static previews).
2. Or run a simple local static server (recommended):
   - With Node (if installed): `npx serve` or `npx http-server` from the project root.
   - With Python 3: `python -m http.server 8000` and open `http://localhost:8000`.

**Development Notes**
- The project is a static prototype; no build step or backend is required.
- Styling is centralized in `styles.css`. JavaScript hooks live in `script.js` and small inline scripts appear in `header.html` for the demo menu.
- Feel free to adapt or extract components for prototypes or design explorations.
