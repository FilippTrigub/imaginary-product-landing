# testdb.testcollection structure

## Overview
- Collection size: 100 documents (read-only snapshot).
- Purpose: Stores simple demonstration records with numeric telemetry and categorical flags.
- Indexes: Default `_id` index only; no secondary indexes present.

## Field reference
- `_id` (ObjectId): Unique document identifier generated by MongoDB.
- `title` (string): Human readable document label following the pattern `"Document <n>"`.
- `bool_field` (boolean): Flag column; current sample shows 55 `true` and 45 `false`.
- `random_value` (double): Floating-point metric ranging from approximately 1.66 to 99.98 with an average near 50.39.
- `normal_value` (double): Floating-point metric ranging from approximately 13.05 to 91.76 with an average near 51.13.

## Example document
```json
{
  "_id": ObjectId("692f1257bd5c05cff6653049"),
  "title": "Document 1",
  "bool_field": true,
  "random_value": 72.2725379712206,
  "normal_value": 56.792592813820164
}
```

## Notes
- The collection name in the database is `testcollection` (no underscore). A similarly named `test_collection` exists but is currently empty.
- No embedded subdocuments or array fields were observed in the sampled records.
- Consider adding indexes if querying frequently on `bool_field` or numeric ranges.
