# Contributing to NovaSphere v2

First off, thank you for considering contributing to NovaSphere v2! It's people like you that make NovaSphere v2 such a great tool.

## Where do I go from here?

If you've noticed a bug or have a question, [search the issue tracker](https://github.com/novasphere/novasphere-v2/issues) to see if someone else in the community has already created a ticket. If not, feel free to [create a new issue](https://github.com/novasphere/novasphere-v2/issues/new).

## Fork & create a branch

If you're looking to contribute to the code, the best way to get started is to fork the repository and create a new branch.

```
git clone https://github.com/your-username/novasphere-v2.git
cd novasphere-v2
git checkout -b my-new-feature
```

## Making your changes

Once you're on your new branch, you can start making your changes. When you're done, you can commit your changes and push them to your fork.

```
git add .
git commit -m "feat: add my new feature"
git push origin my-new-feature
```

## Submitting a pull request

When you're ready to submit your changes, you can create a pull request from your fork to the main repository.

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).
