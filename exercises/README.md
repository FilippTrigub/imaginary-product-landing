# JavaScript Coding Exercises for Beginners

Welcome to the JavaScript Coding Exercises collection! This comprehensive set of exercises is designed to help beginners learn JavaScript fundamentals through hands-on practice.

## üìö Exercise Categories

### 1. **Variables & Data Types** (10 exercises)
Learn about variables, strings, numbers, booleans, and basic operations.

### 2. **Functions** (10 exercises)
Master function declarations, parameters, return values, and arrow functions.

### 3. **Arrays** (10 exercises)
Work with arrays, array methods, and iteration techniques.

### 4. **Objects** (10 exercises)
Understand object creation, properties, methods, and object manipulation.

### 5. **Conditionals** (10 exercises)
Practice if/else statements, switch cases, and logical operators.

### 6. **Loops** (10 exercises)
Learn for loops, while loops, and iteration patterns.

### 7. **DOM Manipulation** (10 exercises)
Interact with HTML elements, events, and dynamic content.

### 8. **String Methods** (10 exercises)
Master string manipulation and common string operations.

### 9. **Array Methods** (10 exercises)
Deep dive into map, filter, reduce, and other array methods.

### 10. **Challenges** (10 exercises)
Combine multiple concepts to solve real-world problems.

## üöÄ Getting Started

1. Navigate to any exercise folder
2. Open the `exercise.js` file
3. Read the instructions in the comments
4. Write your solution
5. Run the tests with `node test.js`
6. Check your solution against `solution.js` if you get stuck

## üìñ How to Use

Each exercise folder contains:
- `exercise.js` - Your workspace with instructions
- `test.js` - Automated tests to verify your solution
- `solution.js` - Reference solution (try not to peek!)
- `README.md` - Detailed explanation and hints

## üéØ Learning Path

**Beginners:** Start with Variables & Data Types, then move through Functions, Arrays, and Objects.

**Intermediate:** Focus on Array Methods, DOM Manipulation, and Challenges.

**Practice Tips:**
- Try to solve each exercise without looking at the solution
- Run tests frequently to check your progress
- Experiment with different approaches
- Read error messages carefully - they're helpful!
